# 前端技术栈方案

更新时间：2026-03-29 14:48:52

## 结论

当前 H5 前端建议直接采用下面这套技术栈：

- `Vue 3`
- `Vite`
- `TypeScript`
- `Pinia`
- `Vue Router`
- `Tailwind CSS`
- `Axios`
- `lucide-vue-next`

## 为什么选这套

- `Vue 3` 适合做中后台和移动端 H5，页面拆分直观，学习成本低。
- `Vite` 启动快，调试效率高，适合快速迭代。
- `TypeScript` 可以提前发现很多接口字段和数据结构错误，后面接后端时更稳。
- `Pinia` 适合管理用户、分身、消息、朋友圈这些共享状态。
- `Vue Router` 方便把聊天、通讯录、发现、我、朋友圈、维度探测拆成独立路由。
- `Tailwind CSS` 适合快速复刻当前 `WMH1.html` 的视觉风格，少写重复样式。
- `Axios` 适合统一封装后端接口和 token。
- `lucide-vue-next` 能继续使用现在页面里那种简洁图标风格。

## 项目分层

- `pages`：页面级逻辑，负责展示和少量交互。
- `components`：通用 UI，例如头像、顶部栏、底部栏、卡片。
- `layouts`：页面外壳，统一顶部和底部导航。
- `stores`：状态管理，先保留本地状态，后续再切服务端状态。
- `services/api`：请求后端接口，后面统一放这里。
- `utils`：放一些可复用的纯函数，例如本地分支预演逻辑。
- `types`：统一管理数据结构，避免前后端字段对不上。

## 从 `WMH1.html` 的迁移方式

- 先把当前 HTML 里的 UI 拆到 `pages` 和 `components`。
- 再把 `localStorage` 里的数据迁移到 `Pinia`，作为过渡层。
- 再把 AI 推演、朋友圈生成、消息回复这些逻辑迁移到 `services`。
- 最后把前端直接调用 AI 的方式改成请求后端接口。
