# 最小接口清单

更新时间：2026-03-29 16:01:16

这份清单的目标不是一次把所有接口做完，而是先保证项目可以“登录、进入自己的世界、再继续用资料和聊天”。

## 公开接口

- `GET /api/health`
- `POST /api/auth/login`

说明：

- `GET /api/health` 用于健康检查。
- `POST /api/auth/login` 用于手机号 + 密码登录，首次登录会自动创建这个用户自己的账号和独立世界。

## 用户与登录

- `GET /api/auth/me`
- `POST /api/auth/logout`

说明：

- `GET /api/auth/me` 用于前端启动时恢复登录态。
- `POST /api/auth/logout` 用于退出当前世界。

## 资料

- `GET /api/profile`
- `PUT /api/profile`

## 分身与推演

- `GET /api/branches`
- `GET /api/branches/:id`
- `POST /api/branches/preview`
- `POST /api/branches`
- `PATCH /api/branches/:id`

说明：

- `preview` 负责根据年份和决策内容返回一个分身草稿。
- `create` 负责把草稿同步成正式分身。

## 聊天

- `GET /api/chats`
- `GET /api/chats/:branchId/messages`
- `POST /api/chats/:branchId/messages`

说明：

- `GET /api/chats` 可以直接给聊天列表页用，返回分身和最近一条消息。
- 前端先发消息到后端，再由后端根据分身身份返回回复。

## 朋友圈

- `GET /api/moments`
- `POST /api/moments`
- `POST /api/moments/:id/like`
- `POST /api/moments/:id/comments`

说明：

- 朋友圈列表、点赞和评论最好先拆成独立接口。
- 后面如果要做分页，这一层最容易扩展。

## 文件上传

- `POST /api/files/upload`

说明：

- 朋友圈图片、头像、封面图都可以走这个接口。
- 现在上传会先写入 `server/uploads/`，再登记数据库。

## AI 任务

- `POST /api/ai/branches/preview`
- `POST /api/ai/moments/generate`
- `POST /api/ai/replies`

说明：

- 这些接口建议只由后端调用第三方 AI。
- 前端不要直接放密钥，也不要直接请求第三方模型。

## 最小上线顺序

1. 先做 `auth`
2. 再做 `profile` 和 `branches`
3. 再做 `messages`
4. 再做 `moments`
5. 最后接 `files` 和 `ai`

## 为什么这样排

- 登录是底座，没有登录就谈不上“每个人自己的世界”。
- 资料和分身是主数据。
- 聊天依赖分身。
- 朋友圈依赖分身和图片。
- AI 和文件上传属于增强能力，可以最后接。
